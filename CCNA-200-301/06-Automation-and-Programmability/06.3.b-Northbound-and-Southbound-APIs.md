# Northbound and Southbound APIs

## Definition
Northbound and southbound APIs describe how an SDN controller communicates with applications above it and network devices below it.

## Simple explanation
Northbound APIs connect apps to the controller. Southbound APIs connect the controller to routers, switches, firewalls, and other infrastructure.

## API directions
| API direction | Connects | Purpose |
|---|---|---|
| Northbound API | Applications to controller | Request services, policy, data, or automation |
| Southbound API | Controller to devices | Configure devices, collect state, control behavior |

## Northbound API examples
- REST APIs used by dashboards and automation tools.
- Intent-based policy requests.
- Inventory and assurance queries.
- Integration with IT service platforms.

## Southbound API examples
- NETCONF
- RESTCONF
- gNMI
- OpenFlow
- CLI/SSH automation
- SNMP for monitoring

## Example flow
1. An automation app sends a request to the controller through a northbound REST API.
2. The controller validates the request and translates it into device-level changes.
3. The controller uses southbound APIs or protocols to configure switches and routers.
4. Devices apply the configuration and report status back.

## Why APIs matter
APIs let software interact with the network in a structured, repeatable way. This is the foundation of automation and programmability.

## Common mistake
Do not confuse API direction with traffic direction. Northbound and southbound describe controller relationships, not whether user traffic is going toward the internet or toward a LAN.

## Exam focus
Northbound APIs face applications. Southbound APIs face network devices.

## Quick review
Northbound APIs connect apps to controllers; southbound APIs connect controllers to infrastructure devices.
