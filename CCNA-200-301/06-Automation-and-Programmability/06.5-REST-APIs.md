# REST APIs

## Definition
A REST API is a web-based interface that lets software interact with a system using standard HTTP methods and structured data.

## Simple explanation
A REST API lets a script or application ask a controller or device for information, create something, update something, or delete something.

## Common HTTP methods
| Method | Purpose | Example use |
|---|---|---|
| GET | Read data | Get device inventory |
| POST | Create data or start an action | Create a new network object |
| PUT | Replace data | Replace a full configuration object |
| PATCH | Modify part of data | Change one setting |
| DELETE | Remove data | Delete an object |

## Common status codes
| Code | Meaning |
|---|---|
| 200 | OK |
| 201 | Created |
| 204 | Success with no response body |
| 400 | Bad request |
| 401 | Unauthorized |
| 403 | Forbidden |
| 404 | Not found |
| 500 | Server error |

## REST API components
- Endpoint: URL path for a resource.
- Method: action such as GET or POST.
- Headers: metadata such as content type or token.
- Body: data sent with the request, often JSON.
- Response: status code and returned data.

## Example request
```text
GET /api/v1/network-device
Host: controller.example.local
Accept: application/json
Authorization: Bearer <token>
```

## Example use in networking
An automation script sends a GET request to a controller REST API to collect all switch serial numbers, software versions, and management IP addresses.

## Authentication
APIs often use tokens, API keys, basic authentication, OAuth, or certificates. Production API access should be protected with HTTPS.

## Exam focus
Know common REST methods and status code meanings. REST APIs commonly exchange JSON over HTTP/HTTPS.

## Quick review
REST APIs let software read and change network systems using HTTP methods and structured data.
