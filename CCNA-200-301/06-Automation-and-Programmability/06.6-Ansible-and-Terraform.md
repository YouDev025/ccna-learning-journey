# Ansible and Terraform

## Definition
Ansible and Terraform are infrastructure automation tools. Ansible is commonly used for configuration management and task automation. Terraform is commonly used for declarative infrastructure provisioning.

## Simple explanation
Ansible is often used to configure things. Terraform is often used to create and manage infrastructure from a desired-state file.

## Ansible basics
- Uses playbooks written in YAML.
- Often agentless and connects with SSH or APIs.
- Good for configuration tasks and operational workflows.
- Uses modules to perform specific actions.
- Can use inventory files to define managed devices.

## Terraform basics
- Uses configuration files written in HCL.
- Describes desired infrastructure state.
- Uses providers to interact with platforms.
- Maintains state to track managed resources.
- Common in cloud and infrastructure-as-code workflows.

## Comparison
| Feature | Ansible | Terraform |
|---|---|---|
| Main style | Procedural/task-oriented | Declarative desired state |
| Common language | YAML | HCL |
| State file | Usually no central state file | Uses state file |
| Common use | Configure devices/services | Provision infrastructure |
| Access method | SSH, APIs, network modules | Provider APIs |

## Ansible playbook example
```yaml
- name: Configure NTP
  hosts: switches
  tasks:
    - name: Set NTP server
      cisco.ios.ios_config:
        lines:
          - ntp server 192.0.2.10
```

## Terraform idea example
```hcl
resource "example_network" "users" {
  name = "users"
  cidr = "192.168.10.0/24"
}
```

## Good practices
- Store automation files in version control.
- Use variables instead of hardcoding values.
- Test in a lab before production.
- Review planned changes before applying.
- Protect credentials and secrets.

## Exam focus
For CCNA, know the high-level difference: Ansible is commonly associated with configuration management and playbooks; Terraform is commonly associated with infrastructure as code and desired state.

## Quick review
Ansible automates configuration tasks; Terraform provisions and manages infrastructure state.
