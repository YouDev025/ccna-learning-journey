# Traditional vs Controller-Based Networks

## Definition
Traditional networks are configured device by device, while controller-based networks use a central controller to manage policy, automation, and network behavior.

## Simple explanation
In a traditional network, each device mostly manages itself. In a controller-based network, a central brain helps coordinate many devices.

## Traditional networking
- Devices are configured individually.
- Control plane and data plane usually run on each device.
- CLI is common for configuration.
- Network-wide consistency depends heavily on manual process.
- Troubleshooting often starts device by device.

## Controller-based networking
- A controller provides centralized management and policy.
- Devices still forward traffic in the data plane.
- APIs are used for automation and integration.
- Policy can be applied consistently across many devices.
- Visibility and assurance tools can show network-wide health.

## Comparison
| Feature | Traditional | Controller-based |
|---|---|---|
| Configuration | Per device | Centralized or template-based |
| Scale | Harder at large size | Easier for many devices |
| Policy | Manually repeated | Centrally defined |
| Visibility | Device-level | Network-wide dashboards |
| Automation | Scripts or manual CLI | APIs, templates, controller workflows |

## Benefits of controller-based networks
- Faster deployment.
- Consistent policy.
- Better visibility.
- Easier automation.
- Centralized monitoring and assurance.

## Tradeoffs
- Controller availability matters.
- Requires learning new workflows.
- Poor policy design can affect many devices quickly.
- Integration and licensing may add complexity.

## Example
A campus controller can create a network policy once and push it to many access switches, instead of manually configuring each switch.

## Exam focus
Controller-based networking centralizes management and policy, but network devices still forward traffic in the data plane.

## Quick review
Traditional networks are device-centric; controller-based networks are centrally managed and easier to automate at scale.
