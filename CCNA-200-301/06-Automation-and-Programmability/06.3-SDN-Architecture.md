# SDN Architecture

## Definition
Software-Defined Networking, or SDN, separates network control from packet forwarding and uses software to centrally manage network behavior.

## Simple explanation
SDN lets software decide how the network should behave, while network devices focus on forwarding traffic according to that policy.

## Main SDN components
| Component | Role |
|---|---|
| Applications | Request network behavior or policy |
| Controller | Translates intent into network instructions |
| Network devices | Forward traffic in the data plane |
| APIs | Connect applications, controllers, and devices |

## SDN layers
- Application layer: tools and apps that express business or network intent.
- Control layer: controller logic that makes decisions and builds policy.
- Infrastructure layer: switches, routers, firewalls, and APs that forward traffic.

## Why SDN is used
- Centralized policy control.
- Better automation.
- Network programmability.
- Faster provisioning.
- Improved visibility.
- Easier integration with applications and cloud systems.

## SDN examples
- Campus network automation with Cisco Catalyst Center.
- Data center overlays using controllers.
- Cloud networks created through APIs.
- WAN path control using SD-WAN controllers.

## Important idea
SDN does not mean there is no hardware. Physical and virtual devices still forward traffic, but the control and management model is more software-driven.

## Exam focus
Know the SDN layers and the role of controllers, applications, APIs, and infrastructure devices.

## Quick review
SDN uses software and controllers to make networks more programmable, centralized, and automated.
