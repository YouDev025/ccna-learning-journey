# AI and ML in Networking

## Definition
Artificial Intelligence and Machine Learning use data and algorithms to detect patterns, make predictions, recommend actions, or automate decisions in network operations.

## Simple explanation
AI/ML tools can study network data and help engineers find problems faster, predict issues, or improve operations.

## Common uses
- Anomaly detection
- Predictive maintenance
- Root-cause analysis
- Traffic pattern analysis
- Security threat detection
- Wireless optimization
- Capacity planning
- Automated recommendations

## AI vs ML
| Term | Meaning |
|---|---|
| AI | Broad field of systems that perform tasks associated with intelligence |
| ML | Subset of AI where systems learn patterns from data |
| Model | Algorithm trained on data to make predictions or classifications |
| Training data | Historical data used to build the model |
| Inference | Using the model to make a prediction or recommendation |

## Networking examples
- Detecting unusual traffic spikes.
- Predicting interface failures from error trends.
- Recommending better wireless channel or power settings.
- Identifying suspicious login behavior.
- Grouping similar incidents to speed troubleshooting.

## Benefits
- Faster issue detection.
- Better visibility across large networks.
- Reduced alert fatigue when tuned well.
- More proactive operations.
- Improved capacity and performance planning.

## Limitations
- AI/ML depends on good data quality.
- Recommendations still need engineering judgment.
- Models can produce false positives or false negatives.
- Poorly tuned automation can create risk.

## Exam focus
For CCNA, understand AI/ML at a high level: pattern detection, prediction, analytics, and network assurance. You do not need to build ML models.

## Quick review
AI and ML help network teams analyze data, detect anomalies, predict issues, and improve operations.
