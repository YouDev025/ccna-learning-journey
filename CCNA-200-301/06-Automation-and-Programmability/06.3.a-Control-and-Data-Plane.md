# Control Plane and Data Plane

## Definition
The control plane makes decisions about where traffic should go. The data plane forwards the actual traffic based on those decisions.

## Simple explanation
The control plane thinks. The data plane moves packets and frames.

## Main planes
| Plane | Purpose | Examples |
|---|---|---|
| Control plane | Builds forwarding information | Routing protocols, STP, ARP/ND decisions |
| Data plane | Forwards user traffic | Switching frames, routing packets |
| Management plane | Configures and monitors devices | SSH, SNMP, syslog, NETCONF, REST APIs |

## Control plane examples
- OSPF forms neighbors and calculates routes.
- STP decides which Layer 2 ports should forward or block.
- A router builds its routing table.
- A switch learns MAC addresses.

## Data plane examples
- A switch forwards a frame out the correct port.
- A router forwards a packet to the next hop.
- An ACL permits or denies traffic as packets pass through.
- QoS marks, queues, or drops traffic.

## Traditional network model
In traditional devices, the control plane and data plane usually exist on each router or switch.

## SDN model
In SDN, much of the control logic can be centralized in a controller, while network devices continue to forward traffic in the data plane.

## Why this matters
Understanding the planes helps with troubleshooting. A routing protocol problem is usually control plane. A forwarding or hardware path issue is usually data plane. An SSH or API access issue is management plane.

## Exam focus
Know the difference between control plane, data plane, and management plane. SDN often centralizes control while devices still forward traffic.

## Quick review
Control plane decides, data plane forwards, and management plane configures and monitors.
