# JSON

## Definition
JSON, or JavaScript Object Notation, is a lightweight structured data format commonly used by APIs and automation tools.

## Simple explanation
JSON is a readable way for systems to exchange data using key-value pairs, lists, strings, numbers, booleans, and nested objects.

## JSON data types
| Type | Example |
|---|---|
| String | `"SW1"` |
| Number | `24` |
| Boolean | `true` or `false` |
| Array | `["SW1", "SW2"]` |
| Object | `{ "hostname": "SW1" }` |
| Null | `null` |

## Example JSON
```json
{
  "hostname": "SW1",
  "managementIp": "192.168.99.10",
  "model": "Catalyst 9300",
  "vlans": [10, 20, 99],
  "reachable": true
}
```

## JSON rules
- Data is written as key-value pairs.
- Keys are strings in double quotes.
- Strings use double quotes.
- Arrays use square brackets.
- Objects use curly braces.
- Commas separate items.
- No trailing comma after the last item.

## JSON in networking
APIs often return device inventory, interface status, VLANs, routes, users, or alerts in JSON format. Automation tools can parse that data and make decisions.

## JSON vs YAML
| Format | Common use |
|---|---|
| JSON | API requests and responses |
| YAML | Human-written automation files, such as Ansible playbooks |

## Common mistake
JSON is strict about syntax. Missing quotes, extra commas, or mismatched braces can break parsing.

## Exam focus
Understand basic JSON structure: objects, arrays, key-value pairs, strings, numbers, booleans, and nesting.

## Quick review
JSON is a structured data format widely used by REST APIs and network automation tools.
