# Servers

## Definition
A server is an endpoint that provides services, applications, or resources to other devices on a network.

## Simple explanation
Clients ask for something, and servers respond. A web browser is a client; the web server provides the page.

## Common server roles
- Web server: hosts websites or web apps.
- DNS server: translates names to IP addresses.
- DHCP server: gives IP settings to clients.
- File server: stores shared files.
- Mail server: sends and receives email.
- Authentication server: verifies user or device identity.
- NTP server: provides accurate time.

## Client-server flow
1. Client sends a request.
2. Network devices forward the traffic.
3. Server processes the request.
4. Server sends a response.
5. Client displays or uses the result.

## Static addressing
Servers often use static IP addresses or DHCP reservations because clients and infrastructure must reliably find them.

## Example
A PC asks DNS for `example.com`. The DNS server replies with an IP address. The PC can then connect to the web server at that address.

## Exam focus
- DNS, DHCP, NTP, and AAA are important network services.
- Servers are endpoints, but they usually provide shared services to many clients.
- Server availability can affect many users at once.

## Quick review
Servers provide services that clients and network devices depend on.
