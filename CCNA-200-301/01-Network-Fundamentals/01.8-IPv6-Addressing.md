# IPv6 Addressing

## Definition
IPv6 is a 128-bit addressing system designed to provide a much larger address space than IPv4.

## IPv6 format
IPv6 uses hexadecimal groups separated by colons:
```text
2001:db8:acad:1:0000:0000:0000:0010
```
Each group is 16 bits. There are 8 groups total.

## Shortening rules
- Leading zeros in a group can be removed.
- One continuous group of all-zero sections can be replaced with `::`.
- `::` can be used only once in an address.

## Example shortening
```text
2001:db8:acad:1:0000:0000:0000:0010
2001:db8:acad:1:0:0:0:10
2001:db8:acad:1::10
```

## Prefix length
IPv6 uses prefix length notation, such as `/64`. Most LAN subnets use `/64`.

## Common IPv6 addresses
- Global unicast: routable on the IPv6 internet.
- Link-local: used on the local link, starts with `fe80::/10`.
- Unique local: private-like internal IPv6, starts with `fc00::/7`.
- Multicast: one-to-many traffic, starts with `ff00::/8`.

## Useful IOS commands
```text
show ipv6 interface brief
show ipv6 route
ping ipv6 <address>
```

## Exam focus
IPv6 does not use broadcast. Neighbor Discovery replaces many IPv4 ARP functions.

## Quick review
IPv6 uses 128-bit hexadecimal addresses and commonly uses `/64` LAN prefixes.
