# Private IPv4

## Definition
Private IPv4 addresses are reserved address ranges used inside private networks. They are not routed directly on the public internet.

## RFC 1918 private ranges
| Range | Prefix | Common use |
|---|---|---|
| 10.0.0.0 - 10.255.255.255 | 10.0.0.0/8 | Large organizations |
| 172.16.0.0 - 172.31.255.255 | 172.16.0.0/12 | Medium networks |
| 192.168.0.0 - 192.168.255.255 | 192.168.0.0/16 | Home and small office networks |

## Why private IPv4 exists
- Conserves public IPv4 addresses.
- Allows organizations to reuse internal address space.
- Works with NAT for internet access.
- Helps separate internal addressing from public addressing.

## NAT/PAT idea
Private hosts usually access the internet through Network Address Translation. With PAT, many internal hosts share one public IP address by using different port numbers.

## Common mistake
`172.16.0.0/12` only includes `172.16.0.0` through `172.31.255.255`. `172.32.1.1` is not private RFC 1918 space.

## Exam focus
Memorize the three RFC 1918 private IPv4 ranges and remember that private addresses usually need NAT/PAT for internet access.

## Quick review
Private IPv4 addresses are used internally and are usually translated before internet access.
