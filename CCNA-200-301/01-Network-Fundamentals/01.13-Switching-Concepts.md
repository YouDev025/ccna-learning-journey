# Switching Concepts

## Definition
Switching is the process of forwarding Ethernet frames within a local network using destination MAC addresses.

## MAC address learning
1. A frame enters a switch port.
2. The switch reads the source MAC address.
3. The switch records the source MAC and incoming port in the MAC address table.
4. The switch uses the destination MAC to decide where to forward the frame.

## Frame forwarding behavior
| Frame type | Switch behavior |
|---|---|
| Known unicast | Forward out the port listed in the MAC table |
| Unknown unicast | Flood out all ports in the VLAN except the incoming port |
| Broadcast | Flood out all ports in the VLAN except the incoming port |
| Multicast | Flood or forward based on multicast features |

## Important concepts
- MAC address table: maps MAC addresses to switch ports.
- VLAN: logical Layer 2 broadcast domain.
- Access port: carries one VLAN.
- Trunk port: carries multiple VLANs using tagging.
- Broadcast domain: devices that receive a broadcast.
- Collision domain: each switch port is its own collision domain.

## Duplex
- Half duplex: send or receive, not both at the same time.
- Full duplex: send and receive at the same time.
Modern switched Ethernet normally uses full duplex.

## Useful IOS commands
```text
show mac address-table
show interfaces status
show vlan brief
show interfaces trunk
clear mac address-table dynamic
```

## Exam focus
Switches learn from source MAC addresses and forward based on destination MAC addresses. Broadcasts stay inside the VLAN unless routed by a Layer 3 device.

## Quick review
Switches make LAN communication efficient by learning MAC addresses and forwarding frames intelligently.
