# On-Premises and Cloud

## Definition
On-premises infrastructure is owned or operated in an organization's physical location or data center. Cloud infrastructure is provided through a cloud service provider and consumed over a network.

## Cloud service models
| Model | You manage | Provider manages |
|---|---|---|
| IaaS | OS, apps, data, network settings | Physical servers, storage, virtualization |
| PaaS | Apps and data | Runtime, OS, infrastructure |
| SaaS | User settings and data usage | Application and infrastructure |

## Deployment models
- Public cloud: shared provider infrastructure.
- Private cloud: cloud-like infrastructure dedicated to one organization.
- Hybrid cloud: combination of on-prem and cloud resources.
- Multi-cloud: use of more than one cloud provider.

## Network concerns
- Connectivity: VPN, dedicated circuits, internet access.
- Security: identity, firewalls, segmentation, encryption.
- Addressing: avoid overlapping private IP ranges.
- Latency: application performance depends on path and distance.

## Example
A company keeps an internal database on-prem but hosts a web application in the cloud. A VPN connects the cloud network to the corporate network.

## Exam focus
Understand on-prem, public cloud, private cloud, hybrid cloud, IaaS, PaaS, and SaaS. Cloud connectivity often depends on VPNs, dedicated circuits, routing, and security policy.

## Quick review
On-prem and cloud are different ways to host services; many real networks use both.
