# Controllers

## Definition
A controller is a centralized system that manages network devices or network behavior. Controllers are common in wireless networks, SDN, and modern automation platforms.

## Simple explanation
Instead of configuring every device one by one, a controller gives you one place to manage policies, settings, and monitoring.

## Common controller types
- Wireless LAN Controller: manages lightweight APs.
- SDN Controller: programs network behavior from a central control point.
- Cisco DNA Center / Catalyst Center: manages campus policy, automation, and assurance.
- Cloud controller: manages devices through a vendor cloud platform.

## What controllers can do
- Push configurations to many devices.
- Monitor device status and client health.
- Enforce policies consistently.
- Coordinate wireless channels and power levels.
- Provide APIs for automation.

## Controller-based networking idea
Traditional networking puts control and data forwarding logic on each device. Controller-based networking centralizes much of the control logic, while network devices still forward the actual traffic.

## Example
A lightweight AP joins a wireless LAN controller. The controller provides SSID settings, security policy, RF management, and monitoring.

## Exam focus
- Control plane decides where traffic should go.
- Data plane forwards actual traffic.
- Management plane is used to configure and monitor devices.
- Controllers improve consistency and automation.

## Quick review
Controllers centralize management and policy, making large networks easier to operate.
