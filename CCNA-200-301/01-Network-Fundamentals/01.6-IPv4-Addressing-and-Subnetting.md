# IPv4 Addressing and Subnetting

## Definition
IPv4 is a 32-bit addressing system used to identify devices and networks. Subnetting divides a larger network into smaller logical networks.

## IPv4 address format
IPv4 addresses use dotted decimal notation:
```text
192.168.1.10
```
Each octet is 8 bits, and the full address is 32 bits.

## Network and host portions
- Network portion: identifies the subnet.
- Host portion: identifies a device inside that subnet.
- Subnet mask or prefix length: shows where the network portion ends.

## Common masks
| Prefix | Subnet mask | Addresses | Usable hosts |
|---|---|---:|---:|
| /24 | 255.255.255.0 | 256 | 254 |
| /25 | 255.255.255.128 | 128 | 126 |
| /26 | 255.255.255.192 | 64 | 62 |
| /27 | 255.255.255.224 | 32 | 30 |
| /28 | 255.255.255.240 | 16 | 14 |
| /29 | 255.255.255.248 | 8 | 6 |
| /30 | 255.255.255.252 | 4 | 2 |

## Example: /24
Address: `192.168.1.10/24`
- Network address: `192.168.1.0`
- Usable range: `192.168.1.1` to `192.168.1.254`
- Broadcast address: `192.168.1.255`

## Example: /26
Network: `192.168.1.0/26`
- Block size: 64
- Subnets: `192.168.1.0`, `.64`, `.128`, `.192`
- First subnet usable range: `192.168.1.1` to `192.168.1.62`
- Broadcast: `192.168.1.63`

## Subnetting steps
1. Identify the prefix length.
2. Find the block size in the interesting octet.
3. List subnet increments.
4. Identify network, usable host range, and broadcast address.
5. Check whether the destination is local or remote.

## Exam focus
Devices in the same subnet communicate directly at Layer 2. Devices in different subnets need a router or Layer 3 switch.

## Quick review
Subnetting controls how IPv4 address space is divided and whether traffic is local or routed.
