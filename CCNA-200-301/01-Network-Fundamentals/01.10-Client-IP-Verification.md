# Client IP Verification

## Definition
Client IP verification means checking that an endpoint has correct IP settings and can communicate with expected network resources.

## Client IP settings to verify
- IP address
- Subnet mask or prefix length
- Default gateway
- DNS server
- DHCP status
- Physical or wireless connection

## Windows commands
```text
ipconfig
ipconfig /all
ping <gateway>
ping <remote-ip>
nslookup <name>
tracert <destination>
```

## Linux/macOS commands
```text
ip address
ip route
ping <gateway>
dig <name>
traceroute <destination>
```

## Troubleshooting flow
1. Confirm the interface is connected.
2. Check whether the IP address is valid for the subnet.
3. Check the default gateway.
4. Ping the default gateway.
5. Ping a remote IP address.
6. Test DNS resolution.
7. Trace the path if remote connectivity fails.

## Common findings
| Symptom | Possible cause |
|---|---|
| Address starts with 169.254 | DHCP failed; APIPA address assigned |
| Can ping gateway, not websites by name | DNS problem |
| Can ping local devices, not remote networks | Gateway or routing problem |
| Wrong subnet mask | Client misjudges local vs remote destinations |

## Exam focus
Separate IP connectivity from DNS. If a client can ping an IP address but cannot resolve a name, the path may be fine and DNS may be the issue.

## Quick review
Verify the client IP address, mask, gateway, and DNS before blaming the wider network.
