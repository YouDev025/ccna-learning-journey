# Virtualization Fundamentals

## Definition
Virtualization allows multiple virtual systems to run on shared physical hardware. In networking, virtualization also creates logical network devices and links.

## Compute virtualization
- Hypervisor: software that runs virtual machines.
- Virtual machine: software-based computer with virtual CPU, memory, disk, and NIC.
- Host: physical server running the hypervisor.
- Guest: VM running on the host.

## Network virtualization
- vSwitch: software switch inside a hypervisor.
- Virtual NIC: network adapter assigned to a VM.
- VLAN: Layer 2 network segmentation.
- Overlay network: virtual network built over an existing underlay network.
- VXLAN: common overlay technology in data centers.

## Benefits
- Better hardware utilization.
- Faster deployment of servers and networks.
- Easier testing and lab environments.
- Supports cloud computing.

## Risks and concerns
- Shared hardware can create shared failure impact.
- Performance depends on host resources.
- Security policies must include virtual networks.
- Troubleshooting requires visibility into physical and virtual layers.

## Example
A physical server runs a hypervisor. Three VMs connect to a virtual switch, and the virtual switch connects to the physical network through the host NIC.

## Exam focus
Virtual switches behave like switches in software. Traffic between VMs on the same host may not leave the physical server.

## Quick review
Virtualization separates logical systems from physical hardware, including virtual networking.
