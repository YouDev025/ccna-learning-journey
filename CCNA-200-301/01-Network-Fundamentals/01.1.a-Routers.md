# Routers

## Definition
A router is a Layer 3 network device that forwards packets between different IP networks. Routers use destination IP addresses and routing tables to decide where to send traffic.

## Simple explanation
A router is like a traffic director between neighborhoods. If a device needs to reach something outside its own local network, it sends the packet to a router.

## Main jobs
- Connect different IPv4 or IPv6 networks.
- Choose the best path using static routes or routing protocols.
- Act as the default gateway for LAN devices.
- Separate broadcast domains.
- Support WAN, VPN, NAT, ACLs, and QoS features.

## Router decision process
1. Receive a frame on an interface.
2. Remove the Layer 2 header.
3. Look at the destination IP address.
4. Search the routing table for the best matching route.
5. Re-encapsulate the packet into a new frame.
6. Forward it out the selected interface.

## Default gateway
A host sends traffic to its default gateway when the destination is outside the local subnet. Without a correct default gateway, local communication may work, but remote network access fails.

## Useful IOS commands
```text
show ip route
show ipv6 route
show ip interface brief
show interfaces
ping
traceroute
```

## Example
PC1 is `192.168.1.10/24` and wants to reach `8.8.8.8`. Because `8.8.8.8` is not in `192.168.1.0/24`, PC1 sends the packet to its default gateway, such as `192.168.1.1`.

## Exam focus
- Routers operate at Layer 3.
- Routers separate broadcast domains.
- Routing tables contain connected, static, and dynamically learned routes.
- Longest prefix match chooses the most specific route.

## Quick review
Routers connect networks and forward packets based on IP addresses.
