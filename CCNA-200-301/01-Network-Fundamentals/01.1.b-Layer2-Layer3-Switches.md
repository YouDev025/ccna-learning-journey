# Layer 2 and Layer 3 Switches

## Definition
A Layer 2 switch forwards Ethernet frames using MAC addresses. A Layer 3 switch can also route packets between IP networks, usually at high speed inside a campus LAN.

## Simple explanation
A Layer 2 switch connects devices in the same local network. A Layer 3 switch can connect local networks to each other, such as different VLANs.

## Layer 2 switch
- Works mainly at OSI Layer 2.
- Learns source MAC addresses.
- Builds a MAC address table.
- Forwards known unicast frames to the correct port.
- Floods broadcasts and unknown unicasts.
- Supports VLANs and trunks.

## Layer 3 switch
- Performs Layer 2 switching and Layer 3 routing.
- Routes between VLANs using SVIs or routed ports.
- Often used as a distribution or core switch.
- Can support static routing and dynamic routing protocols depending on model/license.

## Key comparison
| Feature | Layer 2 switch | Layer 3 switch |
|---|---|---|
| Main forwarding info | MAC address | MAC and IP address |
| Connects same VLAN | Yes | Yes |
| Routes between VLANs | No | Yes |
| Common role | Access layer | Distribution/core |

## Example
A Layer 2 switch can connect PCs in VLAN 10. If VLAN 10 needs to communicate with VLAN 20, a router or Layer 3 switch is needed.

## Useful IOS commands
```text
show mac address-table
show vlan brief
show interfaces trunk
show ip interface brief
show ip route
```

## Exam focus
- VLANs split a switch into multiple logical broadcast domains.
- Trunks carry multiple VLANs between switches.
- Inter-VLAN routing requires a Layer 3 device.

## Quick review
Layer 2 switches forward by MAC address; Layer 3 switches can also route by IP address.
