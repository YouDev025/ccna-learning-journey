# Endpoints

## Definition
An endpoint is any device that creates, uses, sends, or receives data on a network.

## Simple explanation
Endpoints are the devices people or applications actually use. Traffic usually starts at one endpoint and ends at another endpoint or service.

## Examples
- Laptop or desktop computer
- Smartphone or tablet
- Printer
- IP phone
- Security camera
- Server or virtual machine
- IoT sensor

## Endpoint requirements
Most endpoints need:
- An IP address
- A subnet mask or prefix length
- A default gateway
- DNS server information
- Correct physical or wireless connectivity

## Static vs dynamic addressing
| Method | Description | Common use |
|---|---|---|
| Static | Manually configured address | Servers, network devices, printers |
| Dynamic | Learned from DHCP | User PCs, phones, tablets |

## Example scenario
A laptop sends a print job to a printer. The laptop and printer are both endpoints. The switch and router help move the traffic, but they are not the source or final destination of the print job.

## Troubleshooting checklist
- Is the cable or Wi-Fi connection active?
- Does the endpoint have a valid IP address?
- Is the subnet mask correct?
- Is the default gateway correct?
- Can it ping the gateway?
- Can it resolve DNS names?

## Exam focus
- Endpoints are hosts.
- Incorrect IP settings can break communication even when the network devices are working.
- DHCP usually provides IP address, mask, gateway, and DNS information.

## Quick review
Endpoints are where network traffic begins or ends.
