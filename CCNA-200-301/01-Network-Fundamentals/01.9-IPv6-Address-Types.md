# IPv6 Address Types

## Definition
IPv6 address types describe the scope and purpose of an IPv6 address, such as global communication, local-link communication, or multicast.

## Main IPv6 address types
| Type | Prefix | Purpose |
|---|---|---|
| Global unicast | Usually `2000::/3` | Publicly routable IPv6 address |
| Link-local | `fe80::/10` | Communication on the local link |
| Unique local | `fc00::/7` | Private internal IPv6 addressing |
| Multicast | `ff00::/8` | One-to-many communication |
| Loopback | `::1/128` | Local host testing |
| Unspecified | `::/128` | No address assigned yet |

## Important details
- IPv6 has no broadcast address type.
- Multicast is used instead of broadcast for many functions.
- Every IPv6-enabled interface should have a link-local address.
- Link-local addresses are not routed beyond the local segment.

## Anycast
Anycast uses the same address on multiple devices. Traffic is routed to the nearest device according to the routing table.

## Example
A router interface may have global unicast `2001:db8:1:10::1/64` and link-local `fe80::1` at the same time.

## Exam focus
Know `fe80::/10`, `ff00::/8`, `::1`, and that IPv6 does not use broadcast.

## Quick review
IPv6 address types define where an address works and what it is used for.
