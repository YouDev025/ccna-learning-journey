# Spine-Leaf Topology

## Definition
Spine-leaf is a data center topology where every leaf switch connects to every spine switch. Servers connect to leaf switches, and spine switches provide high-speed paths between leaves.

## Simple explanation
Any server can reach any other server through a predictable number of hops: leaf to spine to leaf.

## Components
- Leaf switches: connect servers, storage, firewalls, or edge devices.
- Spine switches: connect leaf switches to each other.
- East-west traffic: traffic between servers inside the data center.
- North-south traffic: traffic entering or leaving the data center.

## Why it is used
- Predictable latency.
- High bandwidth between racks.
- Easy scaling by adding leaf or spine switches.
- Good support for virtualization and cloud-style workloads.

## Design rule
Each leaf should connect to each spine. Leaf switches normally do not connect directly to other leaf switches.

## Exam focus
Spine-leaf is built for modern data centers where server-to-server traffic is common.

## Quick review
Spine-leaf provides scalable, predictable connectivity between data center endpoints.
