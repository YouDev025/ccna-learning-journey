# Wireless Security

## Definition
Wireless security protects WLAN access, authentication, encryption, and client traffic from unauthorized use or interception.

## Simple explanation
Wi-Fi signals travel through the air, so wireless networks need strong authentication and encryption to prevent unwanted access.

## Common wireless security methods
| Method | Notes |
|---|---|
| Open | No encryption; unsafe for private traffic |
| WEP | Legacy and insecure; avoid |
| WPA | Older improvement over WEP; avoid when possible |
| WPA2 | Common secure option using AES/CCMP |
| WPA3 | Newer, stronger option when supported |
| 802.1X | Enterprise authentication with a RADIUS server |

## Personal vs Enterprise
| Mode | Authentication | Common use |
|---|---|---|
| Personal/PSK | Shared password | Home and small office |
| Enterprise | 802.1X credentials/certificates | Business and campus WLANs |

## Wireless threats
- Rogue APs
- Evil twin APs
- Weak pre-shared keys
- Deauthentication attacks
- Guest-to-internal access mistakes
- Misconfigured open networks

## Good practices
- Use WPA2 or WPA3, not WEP.
- Use strong PSKs for small networks.
- Use WPA2/WPA3-Enterprise for business networks.
- Separate guest WLANs from internal networks.
- Disable unused SSIDs.
- Monitor for rogue APs.

## WLAN security dependencies
Wireless security often depends on wired infrastructure too: VLAN mapping, trunks, DHCP, DNS, firewalls, and RADIUS must all work correctly.

## Exam focus
WPA2 uses AES/CCMP and is much stronger than WEP. Enterprise wireless commonly uses 802.1X with RADIUS.

## Quick review
Wireless security uses encryption, authentication, segmentation, and monitoring to protect WLAN access.
