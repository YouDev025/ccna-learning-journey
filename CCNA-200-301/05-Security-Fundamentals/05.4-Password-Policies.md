# Password Policies

## Definition
A password policy defines the rules users and administrators must follow when creating, storing, changing, and protecting passwords.

## Simple explanation
Password policies make passwords harder to guess, steal, or reuse after a compromise.

## Common password policy rules
- Minimum length
- Complexity requirements
- Password history
- Maximum and minimum age
- Account lockout after failed attempts
- Multi-factor authentication for sensitive access
- No shared administrator accounts
- Secure password reset process

## Strong password guidance
A strong password or passphrase should be long, hard to guess, and unique. Length is usually more valuable than complicated but short passwords.

## Network device controls
```text
security passwords min-length 12
login block-for 120 attempts 3 within 60
service password-encryption
enable secret StrongEnableSecret
username admin secret StrongUserSecret
```

## Login protection example
```text
login block-for 120 attempts 3 within 60
```
This blocks login attempts for 120 seconds if 3 failed attempts occur within 60 seconds.

## Good practices
- Use SSH, not Telnet.
- Use `secret` commands instead of weak password storage.
- Use AAA for centralized control.
- Require MFA for remote access where possible.
- Rotate passwords after compromise.
- Do not store passwords in plain text notes or scripts.

## Common mistakes
- Reusing the same password across devices.
- Leaving default credentials in place.
- Using shared admin accounts.
- Relying only on `service password-encryption` for security.
- Allowing unlimited login attempts.

## Exam focus
Know that `service password-encryption` is weak obfuscation, not strong cryptographic protection. Prefer `secret`, SSH, and AAA.

## Quick review
Password policies reduce credential risk by requiring strong, unique, protected, and well-managed passwords.
