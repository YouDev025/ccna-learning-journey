# Local Passwords

## Definition
Local passwords are credentials stored directly on a network device for console, privilege, or remote access.

## Simple explanation
A router or switch can keep its own usernames and passwords. This is useful for small networks or backup access, but centralized AAA is better for larger environments.

## Common password locations
- Console line password
- VTY line password
- Local username database
- Enable password or enable secret

## Basic examples
```text
enable secret StrongEnableSecret
username admin secret StrongUserSecret
line console 0
 login local
line vty 0 15
 login local
 transport input ssh
```

## Enable secret vs enable password
| Command | Security |
|---|---|
| enable password | Weak, stored less securely |
| enable secret | Preferred, stored as a stronger hash |

## Encrypting visible passwords
```text
service password-encryption
```
This hides plain-text passwords in the configuration, but Type 7 encryption is weak. Use `secret` commands when possible.

## SSH requirements
```text
hostname SW1
ip domain-name example.local
crypto key generate rsa
ip ssh version 2
```

## Good practices
- Use `secret` instead of `password` when possible.
- Use SSH instead of Telnet.
- Use unique user accounts.
- Use strong passwords or passphrases.
- Save configuration after changes.
- Use AAA when the network grows.

## Verification commands
```text
show running-config | include username|enable secret
show running-config | section line
show ip ssh
```

## Exam focus
`enable secret` is preferred over `enable password`. SSH is preferred over Telnet because Telnet sends traffic in clear text.

## Quick review
Local passwords protect direct device access, but strong secrets, SSH, and AAA make management access safer.
