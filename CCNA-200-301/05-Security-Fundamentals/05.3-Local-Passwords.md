# Local Passwords

## Definition
Local passwords are credentials stored directly on a device or system to authorize access.

## Simple explanation
A password is a secret that proves a user or administrator is allowed to access a device or service.

## Best practices for local passwords
- Use a long password with a mix of letters, numbers, and symbols
- Avoid common words, birthdays, or predictable patterns
- Never reuse the same password across multiple accounts
- Change passwords after suspected exposure
- Disable or lock accounts after repeated failed login attempts

## Security note
Passwords are often stored as hashes rather than in plain text. A hash is a one-way representation of the password, which improves protection.

## Example
A router administrator uses a strong password to prevent unauthorized configuration changes.

## Exam-style note
Even strong passwords are less effective if users share them or if devices do not enforce lockout policies.

## Quick review
Local passwords protect devices and should be strong, unique, and carefully managed.
