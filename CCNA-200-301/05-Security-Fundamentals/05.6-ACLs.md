# ACLs (Access Control Lists)

## Definition
ACLs are ordered rules used to permit or deny traffic based on specific criteria such as source, destination, protocol, or port.

## Simple explanation
An ACL works like a security checklist. Traffic is compared against the list from top to bottom, and the first matching rule is usually applied.

## Common types
- Standard ACL: filters based mainly on source IP address
- Extended ACL: filters based on source, destination, protocol, and port number

## Typical uses
- Restrict access to internal services
- Block unwanted traffic to or from certain networks
- Apply policy on routers and firewalls

## Example
An extended ACL can deny HTTP traffic from a guest VLAN to the finance server while allowing other traffic.

## Exam-style note
ACL order matters. A broad permit rule placed too early can accidentally allow traffic that should be blocked.

## Quick review
ACLs are powerful policy tools used to control network access and traffic flow.
