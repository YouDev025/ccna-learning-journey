# Security Programs

## Definition
A security program is the organized set of policies, processes, tools, and people used to manage security risk across an organization.

## Simple explanation
A security program is not one firewall or one password rule. It is the ongoing plan for protecting the organization.

## Main parts of a security program
- Security policies and standards
- Risk management
- User awareness training
- Vulnerability management
- Incident response
- Access control processes
- Monitoring and logging
- Business continuity and disaster recovery

## Policy vs standard vs procedure
| Item | Purpose |
|---|---|
| Policy | High-level rule or requirement |
| Standard | Specific required setting or baseline |
| Procedure | Step-by-step instructions |
| Guideline | Recommended practice |

## Common security activities
- Asset inventory
- Patch management
- Backup testing
- Log review
- Phishing training
- Vulnerability scanning
- Access reviews
- Incident response drills

## Risk terms
- Threat: something that can cause harm.
- Vulnerability: weakness that can be exploited.
- Exploit: method used to take advantage of a weakness.
- Impact: damage caused by an event.
- Likelihood: chance that the event occurs.
- Risk: combination of likelihood and impact.

## Example
A company creates a policy requiring MFA for remote access, a standard requiring approved VPN settings, and a procedure for onboarding new VPN users.

## Exam focus
Security is a process, not just a device. Understand policies, procedures, user training, vulnerability management, and incident response at a high level.

## Quick review
Security programs organize people, process, and technology to reduce risk over time.
