# AAA (Authentication, Authorization, and Accounting)

## Definition
AAA is a security framework used to control user access and record activity on network devices and systems.

## Simple explanation
AAA answers three questions: who are you, what are you allowed to do, and what did you do?

## Components of AAA
| Component | Purpose |
|---|---|
| Authentication | Verifies identity |
| Authorization | Determines permissions |
| Accounting | Records activity |

## Common AAA protocols
| Protocol | Common use | Notes |
|---|---|---|
| RADIUS | Network access, VPN, wireless 802.1X | Combines authentication and authorization |
| TACACS+ | Network device administration | Separates authentication, authorization, and accounting |

## Local vs centralized AAA
| Method | Strength | Weakness |
|---|---|---|
| Local accounts | Works without external server | Hard to manage at scale |
| Central AAA | Consistent control and logging | Depends on AAA server reachability |

## Basic AAA concept example
```text
aaa new-model
aaa authentication login default local
username admin secret StrongPassword
```

## TACACS+ idea
```text
tacacs server TAC1
 address ipv4 192.168.100.10
 key SharedSecret
aaa authentication login default group tacacs+ local
aaa authorization exec default group tacacs+ local
```

## Good practices
- Use centralized AAA for larger networks.
- Keep local accounts as a fallback when appropriate.
- Use individual user accounts instead of shared accounts.
- Log administrative activity.
- Protect AAA shared secrets.

## Exam focus
Authentication verifies identity, authorization controls permissions, and accounting records actions. RADIUS is common for network access; TACACS+ is common for device administration.

## Quick review
AAA improves access control and accountability by centralizing login, permissions, and logging.
