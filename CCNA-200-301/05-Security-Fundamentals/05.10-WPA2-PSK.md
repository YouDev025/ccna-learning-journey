# WPA2-PSK

## Definition
WPA2-PSK, also called WPA2-Personal, is a wireless security mode that uses a shared password to authenticate clients and encrypt Wi-Fi traffic.

## Simple explanation
Everyone connecting to the WLAN uses the same Wi-Fi password. This is simple, but the password must be strong and protected.

## Key features
- Uses a pre-shared key instead of individual user accounts.
- Common in homes and small offices.
- Uses AES/CCMP when configured securely.
- Easier to deploy than 802.1X.
- Harder to manage securely at large scale.

## WPA2-PSK strengths
- Simple setup.
- Good security with a strong passphrase.
- Does not require a RADIUS server.
- Widely supported by clients.

## WPA2-PSK weaknesses
- One shared password for many users.
- Password must be changed everywhere if it is leaked.
- No per-user accountability.
- Weak passphrases can be attacked offline if captured.
- Former users may still know the password.

## Good PSK practices
- Use a long, random passphrase.
- Avoid dictionary words and predictable names.
- Change the PSK when staff or trusted users leave.
- Do not reuse the same PSK on guest and internal WLANs.
- Use WPA2/WPA3-Enterprise when individual identity matters.

## PSK vs Enterprise
| Feature | WPA2-PSK | WPA2-Enterprise |
|---|---|---|
| Authentication | Shared password | Individual credentials/certificates |
| Server required | No | RADIUS/AAA server |
| Accountability | Weak | Stronger per-user tracking |
| Best fit | Home/SOHO | Business/campus |

## Exam focus
WPA2-PSK is simple and common for small networks, but enterprise networks usually prefer 802.1X/RADIUS for per-user authentication.

## Quick review
WPA2-PSK secures Wi-Fi with a shared password; strong passphrases and careful sharing are essential.
