# Layer 2 Security

## Definition
Layer 2 security protects the data link layer by securing switch ports and local network traffic from attacks that target the switching infrastructure.

## Simple explanation
Switches are often trusted by default, but attackers can abuse them with spoofing or unauthorized access techniques.

## Common Layer 2 security features
- Port security: limit the number of MAC addresses allowed on a port
- DHCP snooping: prevent rogue DHCP servers from assigning bad addresses
- Dynamic ARP Inspection (DAI): stop ARP spoofing attacks
- BPDU Guard: protect against unauthorized STP changes

## Common threats
- MAC flooding
- ARP spoofing
- Rogue DHCP servers
- STP manipulation

## Example
A switch port is configured to allow only one known MAC address. If someone plugs in an unknown device, the port can shut down or restrict access.

## Exam-style note
Layer 2 security is important because many attacks begin inside the local network and are not visible to traditional edge security tools.

## Quick review
Layer 2 security protects switches and local traffic from common insider and network-based attacks.
