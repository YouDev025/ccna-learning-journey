# IPsec VPN

## Definition
IPsec is a suite of protocols used to secure IP traffic with encryption, integrity, authentication, and anti-replay protection.

## Simple explanation
An IPsec VPN creates a protected tunnel across an untrusted network, such as the internet, so private traffic can travel securely.

## What IPsec provides
- Confidentiality through encryption
- Integrity through hashing/authentication
- Peer authentication
- Anti-replay protection
- Secure site-to-site or remote-access connectivity

## Common VPN types
| Type | Purpose |
|---|---|
| Site-to-site VPN | Connects two networks, such as branch to headquarters |
| Remote-access VPN | Connects individual users to a private network |

## IPsec protocols
| Protocol | Function |
|---|---|
| AH | Authentication and integrity, no encryption |
| ESP | Encryption, integrity, and authentication; most common |
| IKE | Negotiates security parameters and keys |

## IKE phases
- Phase 1: builds a secure management tunnel between peers.
- Phase 2: negotiates IPsec security associations for protected data traffic.

## Tunnel vs transport mode
| Mode | Use |
|---|---|
| Tunnel mode | Protects entire original IP packet; common for site-to-site VPNs |
| Transport mode | Protects payload only; original IP header remains |

## Important terms
- Encryption algorithm: protects confidentiality.
- Hash/integrity algorithm: detects tampering.
- Pre-shared key or certificates: authenticate peers.
- Interesting traffic: traffic that should be protected by the VPN.
- Security Association: negotiated security relationship.

## Example scenario
A branch office uses an IPsec site-to-site VPN over the internet to reach internal servers at headquarters.

## Exam focus
IPsec commonly uses ESP for encrypted VPN traffic. It can provide confidentiality, integrity, authentication, and anti-replay protection.

## Quick review
IPsec VPNs protect IP traffic across untrusted networks, especially for site-to-site and remote-access connectivity.
