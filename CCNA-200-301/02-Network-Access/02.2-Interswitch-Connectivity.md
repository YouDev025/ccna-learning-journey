# Interswitch Connectivity

## Definition
Interswitch connectivity describes how switches connect to each other to carry VLAN traffic, provide redundancy, and scale a LAN.

## Simple explanation
Switches often need links between them. These links may carry one VLAN or many VLANs, depending on the design.

## Common link types
| Link type | Purpose |
|---|---|
| Access link | Carries one VLAN, usually to an endpoint or simple downstream device |
| Trunk link | Carries multiple VLANs between switches or to routers/APs |
| EtherChannel | Bundles multiple physical links into one logical link |

## Trunking basics
A trunk uses VLAN tags so traffic from multiple VLANs can share one physical link. Cisco switches commonly use IEEE 802.1Q trunking.

## Redundancy basics
Multiple interswitch links can improve availability, but Layer 2 loops are dangerous. STP or EtherChannel is used to prevent loop problems.

## Useful commands
```text
show interfaces trunk
show interfaces status
show spanning-tree
show etherchannel summary
show cdp neighbors
show lldp neighbors
```

## Design tips
- Use trunks between switches when multiple VLANs are needed.
- Limit allowed VLANs on trunks when possible.
- Use a non-user native VLAN.
- Bundle parallel links with EtherChannel instead of leaving them as separate Layer 2 paths.

## Exam focus
Trunks, STP, and EtherChannel are core pieces of interswitch connectivity.

## Quick review
Interswitch links connect switches together and often carry multiple VLANs using trunks.
