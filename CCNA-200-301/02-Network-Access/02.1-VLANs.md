# VLANs

## Definition
A VLAN, or Virtual LAN, is a logical Layer 2 network created on a switch. VLANs separate devices into different broadcast domains even when they connect to the same physical switch.

## Simple explanation
A VLAN is like putting switch ports into separate rooms. Devices in the same VLAN can communicate directly, while devices in different VLANs need routing.

## Why VLANs are used
- Reduce broadcast scope.
- Separate users, servers, voice, guests, and management traffic.
- Improve security by grouping devices logically.
- Make networks easier to organize.
- Support flexible designs without changing physical cabling.

## Important VLAN types
| VLAN type | Purpose |
|---|---|
| Data VLAN | Normal user traffic |
| Voice VLAN | IP phone voice traffic |
| Management VLAN | Switch/AP/device management traffic |
| Native VLAN | Untagged traffic on an 802.1Q trunk |
| Default VLAN | VLAN 1 on Cisco switches by default |

## Basic IOS commands
```text
show vlan brief
vlan 10
 name USERS
interface fastEthernet0/1
 switchport mode access
 switchport access vlan 10
```

## Example
PC1 and PC2 are in VLAN 10. PC3 is in VLAN 20. PC1 can communicate with PC2 at Layer 2, but PC1 needs a router or Layer 3 switch to reach PC3.

## Exam focus
A VLAN is a broadcast domain. Inter-VLAN communication requires Layer 3 routing.

## Quick review
VLANs logically separate devices on switches and control the size of Layer 2 broadcast domains.
