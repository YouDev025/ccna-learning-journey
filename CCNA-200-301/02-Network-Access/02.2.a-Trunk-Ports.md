# Trunk Ports

## Definition
A trunk port is a switch port that carries traffic for multiple VLANs over one link.

## Simple explanation
A trunk is like a shared highway for VLANs. Tags identify which VLAN each frame belongs to.

## Common trunk configuration
```text
interface gigabitEthernet0/1
 description Trunk to SW2
 switchport mode trunk
 switchport trunk native vlan 999
 switchport trunk allowed vlan 10,20,30,99
```

## Verification commands
```text
show interfaces trunk
show interfaces switchport
show running-config interface <interface>
show vlan brief
```

## Important trunk concepts
- Trunks commonly connect switches to switches.
- Trunks can also connect switches to routers, firewalls, APs, or virtualization hosts.
- 802.1Q adds a VLAN tag to Ethernet frames.
- Native VLAN traffic is untagged by default.
- Allowed VLAN lists control which VLANs can cross the trunk.

## Common problems
- One side is trunking and the other is access.
- Required VLAN is missing from the allowed VLAN list.
- Native VLAN mismatch.
- VLAN does not exist on one switch.
- DTP negotiation creates an unexpected trunk.

## Exam focus
A trunk carries multiple VLANs. Access ports carry one data VLAN.

## Quick review
Use trunk ports when multiple VLANs need to cross the same link.
