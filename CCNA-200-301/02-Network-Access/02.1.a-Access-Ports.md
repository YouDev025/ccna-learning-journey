# Access Ports

## Definition
An access port is a switch port assigned to one VLAN. It is normally used to connect an endpoint such as a PC, printer, server, AP, or IP phone.

## Simple explanation
An access port belongs to one VLAN. Traffic entering and leaving the endpoint is normally untagged.

## Common access port configuration
```text
interface fastEthernet0/10
 description User PC
 switchport mode access
 switchport access vlan 10
 spanning-tree portfast
```

## Voice VLAN example
```text
interface fastEthernet0/11
 description IP phone and PC
 switchport mode access
 switchport access vlan 10
 switchport voice vlan 20
 spanning-tree portfast
```

## Useful verification commands
```text
show vlan brief
show interfaces switchport
show interfaces status
show running-config interface <interface>
```

## Best practices
- Manually set user-facing ports to access mode.
- Assign unused ports to an unused VLAN and shut them down.
- Use PortFast on endpoint ports.
- Use port security or 802.1X where required.
- Document port descriptions.

## Common mistakes
- Port is left in the wrong VLAN.
- Port is dynamically negotiating trunking when it should be access.
- Endpoint is connected to a shutdown port.
- Voice VLAN is missing for IP phones.

## Exam focus
Access ports carry one data VLAN. Voice VLAN is a special case where an IP phone can tag voice traffic while the attached PC remains in the access VLAN.

## Quick review
Access ports connect endpoints to a single VLAN.
