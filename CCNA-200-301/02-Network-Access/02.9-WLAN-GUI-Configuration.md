# WLAN GUI Configuration

## Definition
WLAN GUI configuration means using a graphical interface, often on a wireless LAN controller or cloud dashboard, to create and manage wireless networks.

## Simple explanation
Instead of typing every wireless setting in the CLI, many WLAN tasks are done through a controller GUI.

## Common GUI configuration items
- WLAN or SSID name
- WLAN enabled/disabled state
- Security method such as WPA2/WPA3
- Authentication method such as PSK or 802.1X
- VLAN or interface mapping
- DHCP behavior
- Radio bands and channel settings
- Client limits or QoS policies

## Typical WLAN creation flow
1. Create a WLAN profile or SSID.
2. Choose security settings.
3. Map the WLAN to a VLAN/interface.
4. Enable the WLAN.
5. Apply settings to the correct AP group/site.
6. Test with a wireless client.

## Security options
| Option | Common use |
|---|---|
| WPA2/WPA3-Personal | Small networks using a shared password |
| WPA2/WPA3-Enterprise | Business networks using 802.1X/RADIUS |
| Guest portal | Guest access with web authentication or acceptance page |

## Verification checks
- SSID is enabled and visible.
- Client can authenticate.
- Client receives an IP address.
- Client lands in the correct VLAN.
- Client can reach required resources.
- Controller/AP health is normal.

## Exam focus
When configuring WLANs in a GUI, pay attention to SSID, security, VLAN/interface mapping, and whether the WLAN is enabled.

## Quick review
A WLAN GUI is commonly used to create SSIDs, set security, map VLANs, and verify client connectivity.
