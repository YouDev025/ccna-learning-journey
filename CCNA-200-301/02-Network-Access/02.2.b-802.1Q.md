# 802.1Q

## Definition
IEEE 802.1Q is the standard VLAN tagging method used on Ethernet trunks. It inserts a tag into an Ethernet frame to identify the VLAN.

## Simple explanation
When many VLANs share one trunk link, 802.1Q labels each frame so the receiving switch knows which VLAN it belongs to.

## How 802.1Q works
- A tag is added to frames crossing a trunk.
- The VLAN ID in the tag identifies the VLAN.
- The receiving switch removes or interprets the tag and forwards the frame in the correct VLAN.
- Native VLAN traffic is untagged by default.

## VLAN ID ranges
| Range | Meaning |
|---|---|
| 1 | Default VLAN on Cisco switches |
| 2-1001 | Normal VLAN range |
| 1002-1005 | Legacy Token Ring/FDDI VLANs |
| 1006-4094 | Extended VLAN range |

## Router-on-a-stick example
```text
interface gigabitEthernet0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0
```

## Native VLAN example
```text
interface gigabitEthernet0/0.99
 encapsulation dot1Q 99 native
 ip address 192.168.99.1 255.255.255.0
```

## Exam focus
802.1Q is the trunking standard to know for CCNA. ISL is old Cisco proprietary trunking and is not used on modern platforms.

## Quick review
802.1Q tags Ethernet frames so multiple VLANs can share one trunk link.
