# Inter-VLAN Routing

## Definition
Inter-VLAN routing is Layer 3 forwarding between different VLANs. Because VLANs are separate broadcast domains, traffic between them must be routed.

## Simple explanation
Switches keep VLANs separate. A router or Layer 3 switch is needed when devices in different VLANs need to communicate.

## Common methods
| Method | Description |
|---|---|
| Router-on-a-stick | One router interface uses subinterfaces and 802.1Q tags |
| Layer 3 switch SVIs | Switch virtual interfaces route between VLANs |
| Routed ports | Physical Layer 3 switch ports act like router interfaces |

## Router-on-a-stick example
```text
interface gigabitEthernet0/0
 no shutdown
interface gigabitEthernet0/0.10
 encapsulation dot1Q 10
 ip address 192.168.10.1 255.255.255.0
interface gigabitEthernet0/0.20
 encapsulation dot1Q 20
 ip address 192.168.20.1 255.255.255.0
```

## Layer 3 switch SVI example
```text
ip routing
interface vlan 10
 ip address 192.168.10.1 255.255.255.0
 no shutdown
interface vlan 20
 ip address 192.168.20.1 255.255.255.0
 no shutdown
```

## Troubleshooting checks
- VLAN exists on the switch.
- Access ports are assigned to the correct VLAN.
- Trunks allow the required VLANs.
- Router subinterfaces or SVIs are up.
- Clients use the correct default gateway.
- `ip routing` is enabled on a Layer 3 switch.

## Exam focus
A host's default gateway is usually the router subinterface or SVI IP address for that VLAN.

## Quick review
Inter-VLAN routing lets separate VLANs communicate through a Layer 3 device.
