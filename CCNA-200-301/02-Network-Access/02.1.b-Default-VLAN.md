# Default VLAN

## Definition
The default VLAN on Cisco switches is VLAN 1. By default, switch ports are members of VLAN 1 until they are assigned to another VLAN.

## Simple explanation
VLAN 1 is the switch's starting VLAN. It exists automatically, but relying on it for normal production traffic is not a good design habit.

## Characteristics of VLAN 1
- Exists by default on Cisco switches.
- Cannot be deleted.
- All switch ports start in VLAN 1 by default.
- Some control-plane protocols may use VLAN 1 by default.
- It is often the default native VLAN on trunks.

## Why avoid using VLAN 1 for users
- It is predictable and commonly targeted.
- It can mix management, user, and control traffic accidentally.
- It makes troubleshooting and segmentation less clear.
- Better designs use dedicated VLANs for users, voice, management, and unused ports.

## Safer practice
```text
vlan 10
 name USERS
vlan 99
 name MANAGEMENT
vlan 999
 name UNUSED
interface range fastEthernet0/10 - 24
 switchport mode access
 switchport access vlan 999
 shutdown
```

## Verification commands
```text
show vlan brief
show interfaces switchport
show interfaces trunk
```

## Exam focus
VLAN 1 is the default VLAN, but production designs usually move user and management traffic to other VLANs.

## Quick review
VLAN 1 exists by default and cannot be deleted, but it should not be the main user or management VLAN in a clean design.
