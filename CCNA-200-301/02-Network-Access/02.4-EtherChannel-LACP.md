# EtherChannel and LACP

## Definition
EtherChannel bundles multiple physical links into one logical link. LACP, or Link Aggregation Control Protocol, is the open standard protocol used to negotiate EtherChannel.

## Simple explanation
Instead of treating several links as separate paths, EtherChannel makes them behave like one bigger logical link.

## Benefits
- Increases bandwidth.
- Provides redundancy if one member link fails.
- Avoids STP blocking individual parallel links.
- Simplifies configuration by using a port-channel interface.

## LACP modes
| Mode | Behavior |
|---|---|
| active | Actively tries to form LACP EtherChannel |
| passive | Waits for the other side to start LACP |
| on | Forces EtherChannel without negotiation |

## Valid combinations
| Side A | Side B | Result |
|---|---|---|
| active | active | Forms LACP |
| active | passive | Forms LACP |
| passive | passive | Does not form |
| on | on | Forms static EtherChannel |
| on | active/passive | Does not form correctly |

## Basic LACP configuration
```text
interface range gigabitEthernet0/1 - 2
 switchport mode trunk
 channel-group 1 mode active
interface port-channel 1
 switchport mode trunk
```

## Verification commands
```text
show etherchannel summary
show interfaces port-channel 1
show interfaces trunk
show spanning-tree
```

## Common mismatch causes
- Different speed or duplex.
- Different trunk/access mode.
- Different native VLAN.
- Different allowed VLAN list.
- Different LACP/static mode.

## Exam focus
Configure settings consistently on member interfaces before bundling them. STP sees the port-channel as one logical link.

## Quick review
EtherChannel bundles links; LACP negotiates the bundle using active and passive modes.
