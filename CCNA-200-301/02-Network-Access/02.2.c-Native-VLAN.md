# Native VLAN

## Definition
The native VLAN is the VLAN used for untagged traffic on an 802.1Q trunk.

## Simple explanation
Most trunk traffic is tagged, but native VLAN traffic is not tagged by default. Both sides of a trunk should agree on the same native VLAN.

## Default behavior
- Cisco switches use VLAN 1 as the default native VLAN.
- Native VLAN frames are untagged by default.
- A native VLAN mismatch can cause traffic leaks or troubleshooting confusion.

## Recommended practice
- Change the native VLAN from VLAN 1.
- Use an unused VLAN as the native VLAN.
- Match the native VLAN on both ends of the trunk.
- Avoid placing user devices in the native VLAN.

## Configuration example
```text
vlan 999
 name NATIVE_UNUSED
interface gigabitEthernet0/1
 switchport mode trunk
 switchport trunk native vlan 999
```

## Verification commands
```text
show interfaces trunk
show interfaces switchport
show cdp neighbors detail
```

## Common problems
- Native VLAN mismatch between switches.
- Untagged traffic appears in an unexpected VLAN.
- VLAN hopping risk is increased by poor trunk/native VLAN design.

## Exam focus
Native VLAN traffic is untagged on an 802.1Q trunk. Both ends of the trunk should use the same native VLAN.

## Quick review
The native VLAN handles untagged trunk traffic and should be changed from VLAN 1 in clean designs.
