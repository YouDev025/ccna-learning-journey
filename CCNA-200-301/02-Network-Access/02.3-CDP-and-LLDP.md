# CDP and LLDP

## Definition
CDP and LLDP are neighbor discovery protocols. They help network devices learn information about directly connected devices.

## Simple explanation
These protocols let a switch or router say, "Here is who I am, what port I am on, and what capabilities I have," to its neighbors.

## CDP vs LLDP
| Protocol | Type | Notes |
|---|---|---|
| CDP | Cisco proprietary | Enabled by default on many Cisco devices |
| LLDP | Open standard | Works across multiple vendors |

## Information they can show
- Neighbor device name
- Local interface
- Neighbor interface
- Device capabilities
- Platform/model
- Management IP address
- Native VLAN or voice VLAN details on some platforms

## Useful commands
```text
show cdp neighbors
show cdp neighbors detail
show lldp neighbors
show lldp neighbors detail
cdp run
lldp run
```

## Security note
Discovery protocols reveal network information. Many organizations disable them on untrusted or user-facing ports.

## Troubleshooting use
CDP/LLDP can quickly confirm whether a cable goes to the expected device and port.

## Exam focus
CDP is Cisco proprietary. LLDP is vendor-neutral. Both only discover directly connected neighbors.

## Quick review
CDP and LLDP are simple but powerful tools for mapping and troubleshooting directly connected devices.
