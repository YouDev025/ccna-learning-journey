# Wireless Architectures

## Definition
Wireless architecture describes how access points, controllers, clients, and wired infrastructure work together to provide WLAN service.

## Simple explanation
A wireless network can be managed one AP at a time, through a controller, or through a cloud platform.

## Common architectures
| Architecture | Description |
|---|---|
| Autonomous AP | Each AP is configured and managed independently |
| Lightweight AP + WLC | APs are centrally controlled by a Wireless LAN Controller |
| Cloud-managed | APs are managed through a vendor cloud dashboard |
| Mesh | APs connect wirelessly to extend coverage where cabling is difficult |

## Split-MAC concept
In controller-based wireless, some AP functions happen on the AP and some happen on the controller. The AP handles real-time radio tasks, while the controller manages policy, authentication, roaming, and configuration.

## CAPWAP
CAPWAP is used between lightweight APs and wireless LAN controllers.
- Control tunnel: AP management and control messages.
- Data tunnel: client traffic, depending on design.

## Local vs centralized forwarding
- Local switching: client data exits at the AP's local switch connection.
- Centralized switching: client data is tunneled back to the controller.

## Exam focus
Know autonomous, lightweight/controller-based, cloud-managed, and mesh wireless designs. Lightweight APs commonly use CAPWAP to join a WLC.

## Quick review
Wireless architecture defines how APs are managed and how wireless client traffic enters the wired network.
