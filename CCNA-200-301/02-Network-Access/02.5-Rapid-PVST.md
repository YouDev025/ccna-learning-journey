# Rapid PVST+

## Definition
Rapid PVST+ is Cisco's per-VLAN implementation of Rapid Spanning Tree Protocol. It prevents Layer 2 loops while allowing faster convergence than older STP.

## Simple explanation
Switch loops can break a network. Rapid PVST+ blocks selected paths so there is one active loop-free Layer 2 path per VLAN.

## Why STP is needed
Layer 2 Ethernet frames do not have a TTL. If a loop exists, broadcasts and unknown unicasts can circulate endlessly and cause a broadcast storm.

## Main roles
| Role | Purpose |
|---|---|
| Root bridge | Central reference switch for the STP tree |
| Root port | Best port toward the root bridge |
| Designated port | Best forwarding port on a segment |
| Alternate port | Backup path that is blocked/discarding |

## Port states in RSTP
- Discarding
- Learning
- Forwarding

## Root bridge selection
The switch with the lowest bridge ID becomes the root bridge. Bridge ID includes priority and MAC address. Lower priority wins.

## Configuration examples
```text
spanning-tree mode rapid-pvst
spanning-tree vlan 10 root primary
spanning-tree vlan 20 priority 4096
```

## Useful commands
```text
show spanning-tree
show spanning-tree vlan 10
show spanning-tree root
```

## Edge port features
- PortFast lets endpoint ports move to forwarding quickly.
- BPDU Guard can shut down a PortFast port if BPDUs are received.

## Exam focus
Rapid PVST+ runs a separate spanning-tree instance per VLAN. The root bridge can be different for different VLANs.

## Quick review
Rapid PVST+ prevents Layer 2 loops and converges faster than classic STP.
