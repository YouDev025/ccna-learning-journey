# Device Management Access

## Definition
Device management access is the method used to configure, monitor, and troubleshoot network devices such as switches, routers, APs, and controllers.

## Common access methods
| Method | Purpose | Security note |
|---|---|---|
| Console | Local out-of-band access | Requires physical access |
| SSH | Secure remote CLI | Preferred over Telnet |
| Telnet | Remote CLI without encryption | Avoid in production |
| HTTPS | Secure web GUI | Common for controllers and appliances |
| SNMP | Monitoring and management | Use SNMPv3 when possible |

## Management IP
A switch needs a management IP address on an SVI if you want to manage it remotely.

```text
interface vlan 99
 ip address 192.168.99.10 255.255.255.0
 no shutdown
ip default-gateway 192.168.99.1
```

## SSH basics
```text
hostname SW1
ip domain-name example.local
crypto key generate rsa
username admin secret StrongPassword
line vty 0 15
 login local
 transport input ssh
```

## Good practices
- Use SSH instead of Telnet.
- Use strong local credentials or AAA.
- Put management traffic in a dedicated VLAN.
- Restrict VTY access with ACLs where possible.
- Save configuration changes.

## Useful commands
```text
show ip interface brief
show running-config | section line vty
show users
show ssh
copy running-config startup-config
```

## Exam focus
For remote switch management, the switch needs a reachable management SVI and a correct default gateway if the manager is on another subnet.

## Quick review
Secure management access uses the right management IP, gateway, credentials, and encrypted protocols such as SSH or HTTPS.
