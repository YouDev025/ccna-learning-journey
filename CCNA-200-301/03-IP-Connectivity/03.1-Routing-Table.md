# Routing Table

## Definition
A routing table is the list of known networks and paths a router or Layer 3 switch uses to forward packets. Each route tells the device where to send traffic for a destination network.

## Simple explanation
The routing table is like a map. When a packet arrives, the router checks the destination IP address and looks for the best matching route.

## Common route sources
| Code | Route source | Meaning |
|---|---|---|
| C | Connected | Network is directly connected to an interface |
| L | Local | IP address assigned to the router interface |
| S | Static | Manually configured route |
| O | OSPF | Learned from OSPF |
| D | EIGRP | Learned from EIGRP |
| R | RIP | Learned from RIP |

## Route entry parts
A route can include:
- Destination network and prefix length
- Administrative distance
- Metric
- Next-hop IP address
- Exit interface
- Route source code

## Example route
```text
S 10.10.20.0/24 [1/0] via 192.168.12.2
```
This means the route to `10.10.20.0/24` is static, has administrative distance 1, metric 0, and uses next hop `192.168.12.2`.

## Longest prefix match
Routers choose the most specific matching route. A `/28` route is more specific than a `/24`, and a `/24` is more specific than a `/16`.

## Administrative distance
Administrative distance decides which route source is trusted when multiple sources advertise the same prefix.

| Route type | Default AD |
|---|---:|
| Connected | 0 |
| Static | 1 |
| EIGRP internal | 90 |
| OSPF | 110 |
| RIP | 120 |

## Useful commands
```text
show ip route
show ipv6 route
show ip route <network>
show ip protocols
```

## Exam focus
A router first uses longest prefix match. Administrative distance is used when the same prefix is learned from different sources. Metric is used when the same routing protocol has multiple paths to the same prefix.

## Quick review
The routing table is the router's forwarding map, built from connected, static, and dynamically learned routes.
