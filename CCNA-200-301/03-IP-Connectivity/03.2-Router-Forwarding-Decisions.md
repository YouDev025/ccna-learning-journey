# Router Forwarding Decisions

## Definition
A router forwarding decision is the process a router uses to choose where to send a packet based on the destination IP address and routing table.

## Simple explanation
When a router receives a packet, it removes the old Layer 2 frame, checks the Layer 3 destination address, chooses the best route, and sends the packet in a new Layer 2 frame.

## Forwarding process
1. Router receives a frame on an interface.
2. Router checks the frame and removes the Layer 2 header.
3. Router reads the destination IP address.
4. Router searches the routing table for the longest prefix match.
5. Router identifies the next hop or exit interface.
6. Router resolves the next-hop Layer 2 address if needed.
7. Router builds a new frame and forwards it out the selected interface.

## Important details
- The source and destination IP addresses usually stay the same end to end.
- The source and destination MAC addresses change at every routed hop.
- TTL is decremented by each router for IPv4.
- If no route exists, the router drops the packet.
- A default route can match traffic when no more specific route exists.

## Default route
A default route is used when no specific route matches.
```text
ip route 0.0.0.0 0.0.0.0 <next-hop>
```

## Connected vs remote destination
| Destination | Forwarding behavior |
|---|---|
| Connected network | Forward directly to destination host MAC |
| Remote network | Forward to next-hop router MAC |
| Unknown network with default route | Forward to default next hop |
| Unknown network without default route | Drop packet |

## Useful commands
```text
show ip route
show ip cef <destination>
show arp
ping
traceroute
```

## Common troubleshooting points
- Missing route.
- Wrong next hop.
- Interface down.
- ARP failure.
- ACL blocking traffic.
- Return path missing.

## Exam focus
Routing decisions are based on destination IP address, longest prefix match, and routing table entries. MAC addresses are rewritten at each routed hop.

## Quick review
Routers forward packets by looking up the destination IP address and sending the packet toward the best next hop.
