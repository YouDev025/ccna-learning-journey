# First-Hop Redundancy

## Definition
First-hop redundancy provides a backup default gateway for hosts. It lets hosts keep using the same gateway IP address even if one router or Layer 3 switch fails.

## Simple explanation
A PC usually has one default gateway. First-hop redundancy makes multiple routers share a virtual gateway so the PC does not need to change settings during a failure.

## Why it is needed
Without first-hop redundancy, a default gateway failure can isolate hosts from remote networks even when another router is available.

## Common protocols
| Protocol | Type | Notes |
|---|---|---|
| HSRP | Cisco proprietary | Uses active and standby routers |
| VRRP | Open standard | Uses master and backup routers |
| GLBP | Cisco proprietary | Can provide load balancing across gateways |

## HSRP concepts
- Virtual IP address: default gateway address used by hosts.
- Virtual MAC address: MAC address associated with the virtual gateway.
- Active router: currently forwarding traffic for the virtual gateway.
- Standby router: takes over if active router fails.
- Priority: higher priority is preferred as active.
- Preemption: allows a higher-priority router to retake active role after recovery.

## HSRP example
```text
interface vlan 10
 ip address 192.168.10.2 255.255.255.0
 standby 10 ip 192.168.10.1
 standby 10 priority 110
 standby 10 preempt
```

A second router or Layer 3 switch might use real IP `192.168.10.3` and the same virtual IP `192.168.10.1` with lower priority.

## Verification commands
```text
show standby brief
show standby
show ip interface brief
ping <virtual-gateway>
```

## Design notes
- Hosts use the virtual IP as their default gateway.
- Each router still has its own real interface IP address.
- Tracking can reduce priority if an upstream link fails.
- FHRP is used at the first hop, not across the whole routed path.

## Common troubleshooting points
- Hosts point to the real router IP instead of virtual IP.
- HSRP group numbers or virtual IPs do not match.
- VLAN or SVI is down.
- Priority/preemption is not configured as expected.
- ACLs or trunk issues block gateway reachability.

## Exam focus
First-hop redundancy protects the default gateway. HSRP is Cisco proprietary; VRRP is open standard. Hosts should use the virtual IP address as their default gateway.

## Quick review
First-hop redundancy keeps hosts connected to remote networks when their primary default-gateway device fails.
