# Single-Area OSPFv2

## Definition
OSPFv2 is a link-state dynamic routing protocol for IPv4. Single-area OSPF means all participating routers are in the same OSPF area, usually area 0.

## Simple explanation
OSPF routers share information about links, build the same network map, and calculate the best paths using cost.

## OSPF basics
- OSPFv2 supports IPv4.
- It uses areas; single-area designs normally use area 0.
- It uses link-state advertisements to describe topology.
- It calculates best paths with SPF, also called Dijkstra's algorithm.
- Metric is cost, based mainly on interface bandwidth.

## Neighbor requirements
Routers must agree on key settings before becoming OSPF neighbors:
- Area ID
- Subnet
- Hello and dead timers
- Authentication, if configured
- Stub area flags, if used
- Compatible network type

## Basic configuration with network statements
```text
router ospf 1
 router-id 1.1.1.1
 network 192.168.12.0 0.0.0.255 area 0
 network 10.1.1.0 0.0.0.255 area 0
```

## Interface-based configuration
```text
interface gigabitEthernet0/0
 ip ospf 1 area 0
```

## Passive interface
Use passive interface on LAN interfaces where no OSPF neighbor should form, but the network should still be advertised.
```text
router ospf 1
 passive-interface gigabitEthernet0/1
```

## Router ID
OSPF chooses router ID in this order:
1. Manually configured router ID.
2. Highest loopback IP address.
3. Highest active physical interface IP address.

## DR and BDR
On multiaccess networks such as Ethernet, OSPF elects a Designated Router and Backup Designated Router to reduce neighbor update overhead.

## Verification commands
```text
show ip ospf neighbor
show ip ospf interface brief
show ip route ospf
show ip protocols
show running-config | section router ospf
```

## Common troubleshooting points
- Area mismatch.
- Wrong wildcard mask.
- Passive interface used accidentally.
- Timer mismatch.
- Authentication mismatch.
- Missing network statement or interface command.
- Interface is down.

## Exam focus
OSPF uses cost as its metric and has administrative distance 110. In CCNA, know neighbor formation, router ID selection, passive interfaces, and basic single-area configuration.

## Quick review
Single-area OSPFv2 dynamically learns IPv4 routes inside one OSPF area, commonly area 0.
